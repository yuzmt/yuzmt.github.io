export 与 export default 的区别

Posted on 2019-08-08

1、export default 和 export 的区别

  • export 与 export default 均可以导出常量、函数、文件、模块等

  • 在一个文件或者模块中，export、import 可以有多个，export default 仅有一个

  • 通过 export 方式导出，在导出是要加上 {}，export default 则不需要

  • 输出单个值的时候，使用 export default

  • 输出多个值的时候，使用 export

  • 这两个不要同时使用

cookie、sessionStorage、localStorage 的区别：

Posted on 2019-07-15

这三种缓存机制还是很常见的，以下是它们主要的区别

一、区别

1、相同点：
  • 都存储在客户端
2、不同点：
  • 存储大小
    • cookie 数据大小一般不超过4k
    • sessionStorage 和 localStorage 一般为5M左右，从这点上来说，比 cookie 大的多
  • 有效时间
    • localStorage 存储持久数据，浏览器关闭后不会自动删除，除非手动删除
    • sessionStorage 数据在前浏览器窗口关闭后自动删除
    • cookie 设置的 cookie 过期时间之前一直有效，即是窗口或浏览器关闭
  • 数据与服务器之间的交互方式
    • cookie 的数据会自动传递到服务器，服务器也可以写 cookie 到客户端
    • sessionStorage 和 localStorage 不会自动把数据发给服务器，仅在本地保存

ES6-箭头函数

Posted on 2019-07-15

在讨论箭头函数之前，我们可以先了解一下，它具有哪些特点：

1、不用写function关键字

2、只能作为一个函数不能new，也不具有原型

3、参数不能重复命名

4、返回值可以不写return，但是有时需要配合{}

5、内部arguments this 由定义时外围最接近一层的非箭头函数的arguments 和 this 决定其值

1、ES5函数 VS ES6箭头函数

function sum(a, b) {
    return a + b;
}
//或者
var sum = function (a,b) {
    return a + b;
}

//ES6的箭头函数
let sum = (a, b) => {
    return  a + b;
}

2、箭头函数可以不写return

let sum = (a, b) => {
    a + b;
    //return undefined //这边如果不写return的话，这边相当于return了一个undefined
}
console.log(sum(10,20))//undefined

要想得到具体的值，我们可以这样写

let sum = (a, b) => a + b;
console.log(sum(10,20)) //30

输出为数组

let sum = (a, b) => [a, b];
console.log(sum(10,20)) //[10,20]

输出为对象

let sum = (a, b) => {a:a,b:b};
console.log(sum(10,20)) //error

系统会将大括号识别成函数体的大括号

let sum = (a,b) => {
    a:a,
    b:b //然而函数里面并没有这样的表达式，所以会报错
}

如果非要输出为对象，我们该怎么办？

let sum = (a, b) => ({a:a,b:b});
//只需将{}外面加一个()，变成表达式即可

let sum = (a, b) => ({
    a: a,
    b: b
});
console.log(sum(10, 20))  //{a: 10, b: 20}

3、参数不能重复命名

在ES6之前,下面的写法在控制台是不会报错的

function a(x,x) {}

在ES6里面，控制台报错，告诉我们重复的名字是不允许出现的

let a = (x,x) => {}
// Duplicate parameter name not allowed in this context

4、只能作为函数，不能new，也不具有原型

下面的例子，控制台告诉我们a不是一个构造函数

let a = (x, y) => {};
new a();
// a is not a constructor at demo.html:87

在控制台输入 a.prototype 返回的值是 undefined

5、内部arguments this 由定义时外围最接近一层的非箭头函数的arguments 和 this 决定其值

（1）arguments
function outer() {
    console.log(arguments)
    let sum = (a, b) => {
        console.log(arguments, a, b);//这里的arguments和outer函数里面arguments是一样的
    }
    sum(1, 2)
}
outer(4, 6, 7)

如果不用箭头函数，内部函数如果需要外部函数的arguments，只能先将arguments赋值给arg这个变量，然后内部函数再调用

function outer() {
    var arg = arguments;
    return function() {
        console.log(arg, arguments)
    }
}
outer(4, 6, 7)(1, 2)

如果是箭头函数，则不必这么麻烦

function outer() {
    return () => {
        console.log(arguments)
    }
}
outer(4, 6, 7)(1, 2)
（2）this

我们需要注意的，箭头函数里面的this在定义函数的时候就已经绑定了，而不是像非箭头函数在执行函数的时候绑定

let sum = () => {
        console.log(this);  // this 指向 window
    }
sum()

//此时外面没有非箭头函数，所以指向window
 var a = 'outerObj';
    let sum = () => {
        console.log(this.a); // outerObj
    }
    var obj = {
        a: 'innerObj',
        fn: sum
    }
    obj.fn();

//如果是正常的函数,谁调用this就指向谁
 var obj = {
        a: 'innerObj',
        fn: sum
    }

    function sum() {
        console.log(this.a); //innerObj
    }
    obj.fn()

下面这个例子里，sum函数为箭头函数，所以就先看它外层是否有非箭头函数，显然这里面是存在的fn函数；那么sum里面的this就是由fn里面的this决定，因此，this.a指向的就是obj里面的a

var a = 'outerObj';
let obj = {
    a: 'innerObj',
    fn() {
        let sum = () => {
            console.log(this.a) //innerObj
        }
        sum();
    }
}
obj.fn()

//在ES5中上面的写法如下：

var a = 'outerObj';
let obj = {
    a: 'innerObj',
    fn:function fn() {
        var _this = this;
        var sum = function sum() {
            console.log(_this.a) //innerObj
        }
        sum();
    }
}
obj.fn()

就算把sum函数return到外部也是不好使的,只需要记住箭头函数里面的this在定义函数的时候就已经绑定了

var a = 'outerObj';
    let obj = {
        a: 'innerObj',
        fn() {
            let sum = () => { 
                console.log(this) //innerObj
            }
            return sum;
        }
    }
    let outerSum = obj.fn()
    outerSum()

ES6-destructuring

Posted on 2019-07-15

destructuring(解构[这个单词本身的意思是破坏])

目的：为了快速拿到某个值，简化代码量

解构对象

let obj = {
    name:'zhang',
    age:18
};
({name,age} = obj) //name : 'zhang'     age:18

//或者  效果是一样的
let {name,age} = obj;

看着变量的名称都是一样的，很难受，我们也可以这样做:

let {name:oName,age:oAge} = obj;
console.log(oName,oAge) //zhang 18

当变量没有同名的属性时，导致取不到值，最后返回undefined

let { bar, foo } = { foo: 'aaa', bar: 'bbb' };
foo // "aaa"
bar // "bbb"

let { baz } = { foo: 'aaa', bar: 'bbb' };
baz // undefined

默认赋值

let obj = {
    name: 'zhang',
    age: 18
};
let {name, age, sex = 'female'} = obj; 
//此时sex的值为'female'

//如果
let obj = {
    name: 'zhang',
    age: 18，
    sex: 'male'
};
let {name, age, sex = 'female'} = obj;
//此时sex的值就为'male'

类似的例子也有：

function sum(a = 10,b = 20) {
    return a + b;
}
sum(1) //返回的是21
//这边只是给    a   赋值了1，并没有给  b   赋值，则拿到 b 的默认值

结构数组（数组的本质是对象）

let arr = [1, 2, 3];
let {
    x,
    y,
    z
} = arr;
console.log(x, y, z)
//undefined undefined undefined

// 别忘了，数组的本质是对象
let {
    0: x,
    1: y,
    2: z
} = arr;
console.log(x, y, z) // 1 2 3

// 如果写成对象的形式觉得比较麻烦的话，其实'化妆'成arr认识的人就好啦
let [x, y, z] = arr;
console.log(x, y, z) // 1 2 3


//我们也可以得到该数组的长度
let {length} = arr;
console.log(length) // 3

价值体现

复杂数据里面需要分开操作的时候，此时 解构 就非常有用

一个对象或者数组里往往会有很多个值，但是这些值又不一定能够全部用上，所以我们可以挑选出我们需要的数据

let arr = [1, 2, 3, name:'zhang'];
let [,,, {name}] = arr;
//将前三个值放空，不要这么多


  let oJson = '{"a":"你", "b":"好", "c":"世","d":"界"}';
    let oRes = JSON.parse(oJson);
    // 获取到数据的两个方式
    // 1、原生js的方式
    let oA = oRes.a;
    let oB = oRes.b;
    let oC = oRes.c;
    let oD = oRes.d;
    // 2、利用解构的方式
    let {
        a,
        b,
        c,
        d
    } = oRes;

对象的解构赋值，可以很方便地讲现有对象的方法，赋值到某个变量

// 例一
let { log, sin, cos } = Math;

// 例二
const { log } = console;
log('hello') // hello

手动封装ajax兼容性写法

Posted on 2019-07-12

ajax 用 JavaScript 以一部的形式操作 xml (现在操作的是 json)

流程原理

其实说到底, ajax 类似于定外卖, 大概的流程如下:

形象比喻——定外卖

  • 第一步，定外卖我们需要一个手机或者一台电脑
  • 第二步，我们要选择在什么平台上去定, 比如, 美团, 饿了么等
  • 第三步，打开APP，寻找商家，再寻找商品；
  • 第四步，下单（可能还需要备注一些信息，加辣，不要葱）
  • 第五步，监听外卖信息，查看我们的外卖是否送出呀，还有多久送达呀等
  • 第六步，开门验货，如果不是自己的订单，可能还需要进一步处理

实际流程

  • 第一步，当然得有一个浏览器

  • 第二步，我们需要一个创建一个 ajax 对象，这里我们需要考虑一下浏览器的兼容性

  • 第三步，ajax.open(method, url, true)，建立对服务器的调用

    • method参数可以是 get 或者 post
    • url 参数可以是相对 url 或者绝对 url
    • 这里的true表示是否异步，所以肯定是true呀

  • 第四步，那就是发送我们的请求，ajax.send()

  • 第五步，通过 onreadystatechange 来监听异步对象的状态

  • 第六步，通过 status 查看状态（200，404等）

1、…展开&收集运算符
此运算符在不同地方使用有不同的功效，可以从写和读两个角度考虑

写：

    function test(...arg){};
    test(1,2,3);-收集作用

    function sum(...arg) {
        let sumNumber = 0;
        // console.log(arg.sort()) //可以直接arg.sort()排序或者arg.pop()等操作
        arg.forEach(function(ele) {
            sumNumber += ele
        });
        return sumNumber;
        console.log(arg) //此时的arg[1,2,3]
    }
    sum(1, 2, 3) //sumNumber = 6
    sum(2, 5, 3, 7) //sumNumber = 17

var arg = [1,2,3];

ES6-let/const

let

1、let来声明变量，不会变量提升

2、let加强了作用域的控制

3、let {} => 块级作用域

临时死区：只要let在{}内定义了变量，就称霸了，不会再找{}外的相同变量

这个{}，如：if(){},for(){}

const

常量 （定义之后，不可变）

    const PI = 20;
如果想 PI = 30 来改变，则会报错

1、存储常量的空间里面的值不可改变

    const PI = {};
PI.name = 'zz';

此时不会报错，因为改变的是{}的空间

2、const剩下内容基本和let差不多

const => {} 也会有临时死区

PS： var let不要混合使用
var尽量用const替代，其次选择let
