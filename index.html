vue-vuex

Posted on 2019-08-26
1、vuex 是什么?

  • vuex 是 Vue 配套的公共数据管理工具, 它可以把一些共享的数据, 保存到 vuex 中, 方便整个程序中的任何组件直接获取或修改我们的公共数据

  • Vuex 是为了保存组件之间共享数据而诞生的, 如果组件之间有要共享的数据, 可以直接挂载到 vuex 中, 而不必通过父子组件之间传值了, 如果组件的数据不需要共享, 此时, 这些不需要共享的私有数据, 没有必要放到 vuex 中

    也就是说, 只有共享的数据, 才有权利放到 vuex 中; 组件内部私有的数据, 只要放到组件的 data 中即可

2、使用

  • 安装包

  • 导入包

  • new Vue.store() 实例, 得到一个数据仓库

    • 如果在组件中想要访问 store 中的数据, 只能通过 this.$store.state.* 来访问, 比如 this.$store.state.count
    • 如果要操作 store 中的 state 值, 只能通过调用 mutation 提供的方法, 才能操作对应的数据, 不推荐 直接操作 state 中的数据, 因为, 万一导致了数据的紊乱, 不能快速定位到错误的原因, 因为, 每个组件都可能有操作数据的方法;
    • 如果组件调用 mutation 中的方法, 只能使用 this.$store.commit('方法名')

    注意: mutation 函数的参数列表中, 最多支持两个参数

    参数1: 是 state 状态

    参数2: 通过 commit 提交过来的参数, 但是这个参数类型是什么并没有什么限定, 我们也可以传一个对象

  • 将 vuex 创建的 store 挂载到 vm 实例上, 只要挂载到了 VM 上, 任何组件都能使用 store 来存取数据

vue-父子组件数据和方法的传递

Posted on 2019-08-17

1、父子组件的传值问题

  • 父组件可以在引用子组件的时候, 通过 属性绑定 (v-bind:) 的形式, 把需要传递给子组件的数据,以属性绑定的形式,传递到子组件的内部,供子组件使用, 需要注意的是, 自定义的属性名不能驼峰的形式

  • 子组件中, 默认 无法访问 到父组件中的 data 上的数据和 methods 中的方法

  • 子组件中的 data 数据, 并不是通过父组件传递过来的, 而是 子组件自身私有的, 比如: 子组件通过 ajax 请求回来的数据都是可以放到 data 身上

  • data 是可读可写, props 中的数据都是 只读, 无法重新赋值

  • 父组件向子组件传递方法

    • 父组件将自身的方法传递给子组件需要通过 事件绑定 的机制: v-on, 方法名是自定义的, 不能写成驼峰形式, 合法即可

1、export default 和 export 的区别

  • export 与 export default 均可以导出常量、函数、文件、模块等

  • 在一个文件或者模块中，export、import 可以有多个，export default 仅有一个

  • 这两个不要同时使用

ES6-箭头函数

Posted on 2019-07-15

在讨论箭头函数之前，我们可以先了解一下，它具有哪些特点：

1、不用写function关键字

2、只能作为一个函数不能new，也不具有原型

3、参数不能重复命名

4、返回值可以不写return，但是有时需要配合{}

5、内部arguments this 由定义时外围最接近一层的非箭头函数的arguments 和 this 决定其值

1、ES5函数 VS ES6箭头函数

function sum(a, b) {
    return a + b;
}
//或者
var sum = function (a,b) {
    return a + b;
}

//ES6的箭头函数
let sum = (a, b) => {
    return  a + b;
}

2、箭头函数可以不写return

let sum = (a, b) => {
    a + b;
    //return undefined //这边如果不写return的话，这边相当于return了一个undefined
}
console.log(sum(10,20))//undefined

要想得到具体的值，我们可以这样写

let sum = (a, b) => a + b;
console.log(sum(10,20)) //30

输出为数组

let sum = (a, b) => [a, b];
console.log(sum(10,20)) //[10,20]

输出为对象

let sum = (a, b) => {a:a,b:b};
console.log(sum(10,20)) //error

系统会将大括号识别成函数体的大括号

let sum = (a,b) => {
    a:a,
    b:b //然而函数里面并没有这样的表达式，所以会报错
}

如果非要输出为对象，我们该怎么办？

let sum = (a, b) => ({a:a,b:b});
//只需将{}外面加一个()，变成表达式即可

let sum = (a, b) => ({
    a: a,
    b: b
});
console.log(sum(10, 20))  //{a: 10, b: 20}

3、参数不能重复命名

在ES6之前,下面的写法在控制台是不会报错的

function a(x,x) {}

在ES6里面，控制台报错，告诉我们重复的名字是不允许出现的

let a = (x,x) => {}
// Duplicate parameter name not allowed in this context

4、只能作为函数，不能new，也不具有原型

下面的例子，控制台告诉我们a不是一个构造函数

let a = (x, y) => {};
new a();
// a is not a constructor at demo.html:87

在控制台输入 a.prototype 返回的值是 undefined

5、内部arguments this 由定义时外围最接近一层的非箭头函数的arguments 和 this 决定其值

（1）arguments
function outer() {
    console.log(arguments)
    let sum = (a, b) => {
        console.log(arguments, a, b);//这里的arguments和outer函数里面arguments是一样的
    }
    sum(1, 2)
}
outer(4, 6, 7)

如果不用箭头函数，内部函数如果需要外部函数的arguments，只能先将arguments赋值给arg这个变量，然后内部函数再调用

function outer() {
    var arg = arguments;
    return function() {
        console.log(arg, arguments)
    }
}
outer(4, 6, 7)(1, 2)

如果是箭头函数，则不必这么麻烦

function outer() {
    return () => {
        console.log(arguments)
    }
}
outer(4, 6, 7)(1, 2)
（2）this

我们需要注意的，箭头函数里面的this在定义函数的时候就已经绑定了，而不是像非箭头函数在执行函数的时候绑定

let sum = () => {
        console.log(this);  // this 指向 window
    }
sum()

//此时外面没有非箭头函数，所以指向window
 var a = 'outerObj';
    let sum = () => {
        console.log(this.a); // outerObj
    }
    var obj = {
        a: 'innerObj',
        fn: sum
    }
    obj.fn();

//如果是正常的函数,谁调用this就指向谁
 var obj = {
        a: 'innerObj',
        fn: sum
    }

    function sum() {
        console.log(this.a); //innerObj
    }
    obj.fn()

下面这个例子里，sum函数为箭头函数，所以就先看它外层是否有非箭头函数，显然这里面是存在的fn函数；那么sum里面的this就是由fn里面的this决定，因此，this.a指向的就是obj里面的a

var a = 'outerObj';
let obj = {
    a: 'innerObj',
    fn() {
        let sum = () => {
            console.log(this.a) //innerObj
        }
        sum();
    }
}
obj.fn()

//在ES5中上面的写法如下：

var a = 'outerObj';
let obj = {
    a: 'innerObj',
    fn:function fn() {
        var _this = this;
        var sum = function sum() {
            console.log(_this.a) //innerObj
        }
        sum();
    }
}
obj.fn()

就算把sum函数return到外部也是不好使的,只需要记住箭头函数里面的this在定义函数的时候就已经绑定了

var a = 'outerObj';
    let obj = {
        a: 'innerObj',
        fn() {
            let sum = () => { 
                console.log(this) //innerObj
            }
            return sum;
        }
    }
    let outerSum = obj.fn()
    outerSum()
