nodeJS 简介

​ node.js 是基于 JavaScript 语言和 V8 引擎的 web 服务器项目, 所以我们可以直接使用 JavaScript 来搭建服务器, 就这一点来说, 我们的学习成本大大降低了

nodeJS 的特点

  • node.js 是一个 JavaScript 运行环境
  • 依赖于 chrome V8 引擎进行代码解释
  • 事件驱动
  • 非阻塞 I/O [input/output(输入/输出)]
  • 轻量、可伸缩、适用于实时数据交互应用
  • 单进程、单线程

nodeJS 能为我们解决什么问题

  • 并发连接

    并发: 是指在一个系统中，拥有多个计算，这些计算有同时执行的特性，而且他们之间有着潜在的交互

    我们只看这个概念会头很大, 我在一篇博客文章上看到了比较形象的故事解释

    “我们吃饭吃到一半的时候, 突然来电话了, 我们停了下来去接电话, 接完电话之后继续吃饭”, 这表示支持并发

  • I/O 阻塞

    A 在餐厅吃饭，就这一个座位，A 点完餐后，坐在这享受美食。
    然而这时候来了个 B，点餐后想坐下吃饭，而这时候 A 还没吃完呢, B 只能在旁边眼巴巴的看着 A 吃。得等 A 吃完，走了之后，B 才能坐下来吃饭。而这等待的过程呢，就可以叫做阻塞

    node.js 遇到 I/O 事件, 并不会先处理, 而是先放到事件队列中, 主线程怡然继续往下执行, 利用异步操作来处理事件队列中的事情

    大概的过程如下:

    • 每个 Node.js 进程只有一个主线程在执行程序代码
    • 当用户的网络请求或者其它的异步操作到来时，Node.js 都会把它放到“事件队列”之中，并不会立即执行它，代码就不会被阻塞，主线程继续往下走，直到主线程代码执行完毕
    • 当主线程代码执行完毕完成后，通过事件循环机制，从“事件队列”的开头取出一个事件，从线程池中分配一个线程去执行这个事件，接下来继续取出第二个事件，再从线程池中分配一个线程去执行，一直执行到事件队列的尾部。期间主线程不断的检查事件队列中是否有未执行的事件，直到事件队列中所有事件都执行完，此后每当有新的事件加入到事件队列中，都会通知主线程按顺序取出交代码循环处理。当有事件执行完毕后，会通知主线程，主线程执行回调，线程归还给线程池
