react脚手架相关问题

Posted on 2019-08-07 | 次阅读

一、脚手架创建 react 项目的时候, 在环境以及安装包都没有问题的前提下, 报错?!!!

输入 create-react-app my-demo 之后, 报错且 my-demo 文件夹里面只有一个 yarn-error.log 文件

原因: 因为之前设置了 npm 的注册 URL 为淘宝的, 所以有这个错误(一脸懵*)

现在还原为 https://registry.npmjs.org/ 就可以了

命令如下:

npm config set registry https://registry.npmjs.org/

然后正常执行就可以了

二、用 react 脚手架构建项目, 执行 yarn eject 暴露 webpack 的配置文件时报错????

![yarn eject](react脚手架/yarn eject.png)

This git repository has untracked files or uncommitted changes:

这个git存储库有未跟踪的文件或未提交的更改

解决办法:

git add . (add 和 . 之间有空格的)

git commit -m "init"

yarn eject

三、react 脚手架创建项目, 编译 less 文件的一系列操作!!!!

1、创建项目并运行

create-react-app react-less

cd react-less

yarn start

2、暴露需要配置的文件 config (这个操作时不可逆的)

yarn eject

有时候, 执行上面这个命令没有效果, 会提示 This git repository has untracked files or uncommitted changes , 此时执行如下命令即可

git add .

git commit -m "init"

yarn eject

我们也可以不暴露文件, 找到 node_modules/react-scripts/config/webpack.config.js 同样可以;

值得注意的是, 这种方式在安装依赖的时候, 一定要切换到 react-scripts 这个目录下下载

3、安装 less less-loader 依赖

yarn add less less-loader

4、修改 webpack 的配置

在 webpack.config.js 文件找到关于 css 文件的加载规则

修改下面两个地方

每次修改完配置文件都需要 yarn start

还没完, 扶我起来~(这种情况有时并没有出现)

当我们安装好依赖, 修改好配置, 有时它会接着报错

我们的常规操作就是安装这个缺失的模块, 但是会源源不断的报错, 我们就会源源不断的安装, 很难搞的….

所以

npm install

解决一切

继续 (有时会出现)

![babel runtime](react脚手架/babel runtime.png)

执行这个命令

npm add @babel/runtime

此时, 一切就 OK 啦!
