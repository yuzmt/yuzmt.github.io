ES6-let/const

Posted on 2019-07-11 | 次阅读

let

1、let来声明变量，不会变量提升

2、let加强了作用域的控制

3、let {} => 块级作用域

临时死区：只要let在{}内定义了变量，就称霸了，不会再找{}外的相同变量

这个{}，如：if(){},for(){}

const

常量 （定义之后，不可变）

    const PI = 20;
如果想 PI = 30 来改变，则会报错

1、存储常量的空间里面的值不可改变

    const PI = {};
PI.name = 'zz';

此时不会报错，因为改变的是{}的空间

2、const剩下内容基本和let差不多

const => {} 也会有临时死区

PS： var let不要混合使用
var尽量用const替代，其次选择let
